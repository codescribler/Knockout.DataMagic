Welcome to DataMagic.

Supper simple 3-way data binding for your knockout view models. A fast and effective way to get your data into a database and get you app up and running in no time. Simply point your view model at a database like Firebase or an api url and DataMagic will save your changes as they happen!

Enough of the chat, take a look at the todo demo!

The code is very simple. Assuming you have a Firebase end point ready to go, just chuck your view model at datamagic and watch as changes are stored at your database. This is a simple example taken from the unit tests:



    // A very simple view model
    var viewModel = {
        name  : ko.observable('Daniel'),
        age   : ko.observable(35)
    };

    // Tell datamagic where your firebase end point is
    var options = {
        baseUrl     : "http://www.example.com/"
    };

    var fireWire = new datamagic.FireWire(options);

    // Initialise the provider
    var provider = new datamagic.dm(
        {
            viewModel: viewModel,           // The view model to watch
            wire: fireWire,                 // The wire to send the data over
        });

    // Start watching for changes in the model to save!
    provider.start();

This is an early version of datamagic, however it does work but needs a few more features and tests to get it ready for production.
